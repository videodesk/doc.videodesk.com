Cobrowse masking

Mask personnal identifiable information (PII) from the agent while cobrowsing.

What is the Cobrowse Masking feature ?

While in cobrowse the agent can have a look at the webuser's page. Some of the content may be sensitive data. The Agent should not see nor be able to interfere with this specific content.

Use a specific class to cloak DOM elements to the cobrowse engine.

Please try our demo

How to enable Cobrowse Masking

VideoDesk's cobrowse masking is enabled by default, this is part of the PCI certification. By default, the cobrows tool will ignore:

  • Every input set as autocomplete=off.
  • Every input set as type=password.
  • Every element set as .videodesk-cobrowse-mask.

Usage

Ignoring zones or fields

To ignore a field or a zone, please add the videodesk-cobrowse-mask class to any element.

In this case, the word 'p4ssw0rd' will be ignored.

  <div>
    Please note that your password is
    <span class="videodesk-cobrowse-mask">p4ssw0rd</span>
  </div>

In this case, the whole address will be ignored.

  <div>
    Our product will be shipped at
    <address class="videodesk-cobrowse-mask">
      Mozilla Foundation<br>
      1981 Landings Drive<br>
      Building K<br>
      Mountain View, CA 94043-0801<br>
      USA
    </address>
  </div>

You can add this class to any form element too. In this case, only the pet question will be available to the agent.

  <p>Who did you vote for ?</p>
  <select class="videodesk-cobrowse-mask">
    <option>George W Bush</option>
    <option>Al Gore</option>
    <option>I did not vote</option>
  </div>
  <p>Why ?</p>
  <input type="text" class="videodesk-cobrowse-mask">
  <p>What is your favorite pet ?</p>
  <input type="text">
  <input type="submit" class="videodesk-cobrowse-mask" value="Send form">

Advertise to the user

To reassure the webuser, you may want to explicitly point zones that are not available to the agent. To achieve this, use the .videodesk-cobrowse-masking-marker and .videodesk-cobrowse-masking-marker-enabled classes.

  .videodesk-cobrowse-masking-marker {
    display: none;
  }
  .videodesk-cobrowse-masking-marker-enabled {
    display: block;
  }

In your html it should be looking like this

  <input type="text" class="videodesk-cobrowse-mask">
  <p class="videodesk-cobrowse-masking-marker">
    The Agent will not be able to read the value of the
    above field
  </p>

Custom rules

If you want to add more custom masking rules, please just add this JavaScript snippet to your page or any of your script: on a _vdk.cobrowse_masking object, set the URL of the page and then an array of objects that you would like to hide.

  _vdk.cobrowse_masking = {
    "http://www.mywebsite.com/checkout.php": ["#private"]
  };
