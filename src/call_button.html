Call button

The call button allows you to make calls directly to a specific agent or an agent pool (call-center behavior).

This is an UI abstraction component

Abstraction mode must be enabled to use this feature. To learn more about this, please refer to the user interface abstraction documentation.

The _vdk.call() method

Use the _vdk.call method to make a call, here is a sample usage:


_vdk.call("text", 0);

This can obviously not be called before the VideoDesk script has been loaded. It would be a great idea to check availability before making a call.

Anatomy

The _vdk.call method takes 2 arguments

  • call_mode (string) a call mode, "video" or "text".
  • agent_uid (string) a hash to identify an agent or (int) 0 to call any of the available agents.

Call mode

If your VideoDesk account allows audio chat and if the target agent (or pool) is available, you can initiate a "audio" call, let's make one:


_vdk.call("audio", 0);

If your VideoDesk account allows video chat and if the target agent (or pool) is available, you can initiate a "video" call, let's make one:


_vdk.call("video", 0);

Agent uid

Each agent is identified across websites with an unique agent_uid. If you have a multi-site account, his/her display name may change as well as his/her avatar picture, and his/her availability but his/her agent_uid will remain the same.

You can use a static agent uid (see what is my agent_uid) or gather this from the availability object.

agent_uid is a string containing a hash "03232de131b797f22b90ca296cbd4bd1". This code sample calls Gaspard on the http://www.videodesk.com website (if I am available).


_vdk.call("text", "03232de131b797f22b90ca296cbd4bd1");

If you do not have individual agents' availability or hash identifyers (because you may be in call-center mode), you can call a pool and the call will be distributed among available agents who are able to answer in this call_mode. To do this, simply set the agent_uid to 0 (integer)
